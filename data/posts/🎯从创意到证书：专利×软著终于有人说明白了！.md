# 1、专利、软著都是什么？
大家常听说**“专利”**和**“软著”**，和文章的著作权一样，这就是来证明你是原创/开发这个内容的人。在实操上，可更详细的分为四类：**发明专利、实用新型专利、外观设计专利、计算机软件著作权（软著****）**。

<!-- 这是一张图片，ocr 内容为： -->
![](image/1763024818125-c340c205-6f54-405f-986c-e1a0b0aa8dcd-07133d6a04.png)

**专利权**（依据《专利法》取得）用于保护**技术方案或产品外观**，并赋予**排他性**：保护技术、方法、结构等等，<font style="color:#DF2A3F;">不允许他人进行</font>**<font style="color:#DF2A3F;">制造、使用、许诺销售、销售、进口</font>**。其中，发明专利保护**技术方案**（期限20年），实用新型保护产品的**结构/形状等技术方案**（期限10年），外观设计保护**产品外观设计**（期限15年）。

> **举个例子(****❤**** ω ****❤****)**
>
> + **发明专利**：_一种基于深度学习的医学影像病灶检测方法及系统_（保护算法/流程与实现思路的技术方案）
> + **实用新型专利**：_一种可快速更换电池的无人机电池卡扣机构_（保护具体结构改进的装置）
> + **外观设计专利**：_智能手环外观设计_（保护产品外形/图案/色彩组合的视觉设计）
>
> **为什么有保护期限？**因为发明始终是为了人类社会进步，不可能永远独属于某人，所以只保护一段时间的利益，过后就会公开。
>
> **如果所有人都等保护期过了再去用呢？**那说明发明还不够好用，真正好用的发明带来的经济价值远超专利授权费用。
>

**软件著作权**（依据《著作权法》《计算机软件保护条例》）保护**软件的表达形式**——如源代码、目标代码、文档与界面等，**不延及思想、算法或技术方案本身**。软著登记用于**确权与取证**，是你用某种公开技术制作的产品（表现）。  如果别人自己从零编写代码、独立撰写文档、重新绘制界面素材；即使功能等效或相似，也不算作侵权。 

> **举个例子(****❤**** ω ****❤****)**
>
> + **计算机软件著作权（软著）**：_智能健康管理系统 V1.0_（保护源代码与文档等表达，不及于算法思想）
>
> **源代码**：开发者写的**可读文本**，如 `.py/.java/.c` 里的函数、注释、变量名、文件与模块的组织方式。
>
> **目标代码**：把源代码**翻译给机器运行**后的产物，通常是 `.exe/.o/.so`，或 Java 的 `.class`、Python 的 `.pyc` 等，**人类难以直接阅读**。  
>

> + **印花税**：制发证书时缴纳的税费。
> + **年费**：授权当年起按年度缴纳；逾期未交会**权利终止**，所以要每年都花钱。

>
> 这组权利要求的结构完整：以**方法**为主权（权利要求1），通过若干从属项（2–6）细化关键模块与数据流，再以**训练方法**（7）、“**装置**”（8）和“**计算机设备/存储介质**”（9、10）对应覆盖软件部署与程序产品，构成“方法—装置—程序—介质”的全套保护链条。
>
> 就**独立权1**而言，技术方案的核心是“**影像+临床文本的多模态分割**”流水线：获取直肠癌病灶影像及其**对应临床文本**→经“**文本多尺度特征提取模块**”得到文本多尺度特征→与影像一起送入含**四级多模态特征提取与融合模块**的**特征编码器**→输出的多尺度融合特征再经**尺度注意力模块**重加权→进入**特征解码器**重建→经**分割头**产出病灶分割结果。权1使用“**包括**”表述（开放式用语），但同时**限定了若干必备要素**：必须同时使用临床文本信息；编码器被明指为“四个”层级；解码前须经历“尺度注意力模块”。这些限定共同刻画了“文本多尺度—多级图文融合—尺度注意力—解码分割”的主干思路。
>
> **从属权2–6**把实现细节逐层坐实，提供授权与稳定性的“备份层级”。权2把“文本多尺度特征”落到**文本编码器+多层感知机+状态空间模型**的并行多尺度管线；权3规定编码器的**分层下采样**与“第1至第4”级融合特征的生成路径；权4把首级融合模块细化为**视觉状态空间提取、多核空间注意力、3×3卷积、交叉注意力、MLP、通道注意力**的串并联组合并给出具体的数据流；权5进一步给出“视觉状态空间提取”的**标准化—全连接—深度可分离卷积—二维状态空间**序列；权6则将**尺度注意力与解码器的逐级上采样相加**关系说明清楚。整体看，这些从属项把“为什么能更好分割”的可实施性与技术效果支撑了起来。
>
> **权7**覆盖**训练流程**：含数据增强、将“分割结果+文本编码特征”代入**预置损失函数**反向更新、以“更新次数达阈值”判定收敛。该项可用于约束“仅训练他人模型”的行为，但实践维权取证难度较方法/装置更高，因此与权8–10配合更有现实效果。**权8**将方法模块化为**获取/输入/提取/编码/尺度/解码/分割**等功能模块的装置结构，便于覆盖**在端侧部署的整机或SDK**；**权9/10**是典型的**计算机设备/介质**项，确保对“可执行程序”和“模型权重+推理代码”的分发形成产品层面的保护闭环。
>
> 就**保护范围与可规避点**看，当前权1对**疾病对象**限定为“直肠癌”，对**编码器层级**限定为“四级”，对**是否必须引入临床文本**作了刚性要求。这些都是**潜在设计绕开路径**：例如改为“胃癌/宫颈癌”等其他部位，或仍做图文多模态但把编码器改成“三/五级”，或仅用影像不引入临床文本，均可能避开权1；不过若竞争方案虽改动层数但在功能—方式—效果上与“四级多模态融合”实质等同，仍可能触及**等同侵权**的判断空间。相应地，本案已通过从属项把**文本状态空间建模、多核空间注意力、交叉注意力、通道注意力**等“关键算子组合”写实，为出现“只改层数不改机理”的方案准备了更窄而稳的落脚点。
>
> 综上，这套权利要求以“文本引导的多尺度图文融合+尺度注意力重加权”为发明要点，方法—装置—程序—介质覆盖较全，**授权逻辑扎实**。若后续还要增强抗规避能力，可考虑在继续分案/后续申请中：将疾病对象从“直肠癌”推广为“消化道肿瘤/实体瘤/医学影像病灶”，将“四个模块”表述为“至少两个/若干层级”，并对“尺度注意力”的实现泛化为“通道或空间注意力及其组合”，形成“广—专—窄”多圈层布局。
>

**· ****《说明书》**

为什么要分为初步审查和实质审查？

+ **初步审查（形式/程序）**：查**有没有、对不对**——文件是否齐全、格式是否合规、信息是否一致、分类是否正确、是否属于可授予客体；不合格→补正或视为撤回。
+ **实质审查（专利性）**：查**值不值得授予**——检索对比文献，判断新颖性/创造性/实用性，考察说明书是否充分公开、权利要求是否清楚并有据可依；往复答复直至授权或驳回。

把审查拆成**初步审查**和**实质审查**，本质是把“材料是否合格”和“技术是否值得授予”两件事分开做。先用初步审查把低成本的问题一次清掉：材料是否齐全、格式是否规范、图文是否一致、权利人信息是否准确、是否属于可授予的客体、如何分类与公开等。只有这些都达标，文档才能被公开、被检索，也避免把不合格案件带入后续环节浪费资源。等程序面干净了，再进入实质审查，审查员才投入大量检索与比对，围绕新颖性、创造性、实用性以及说明书支撑、权利要求清楚等实体标准做“值不值得给证”的判断。

# 中山大学专利发表流程
**如果需要走学校报销，必须走学校的专利发表流程**

在提交国家知识产权局专利申请前，需按照学校管理规定，完成内部申请提案流程，确保专利归属、权利分配、申请主体等信息明确。

1、科研人员通过“科研管理协同创新服务平台”进入“专利提案申请”板块。（**学生无法进入该平台，只有老师才有权限进入**）

2、点击“专利提案申请”，完善专利基本信息，上传相应附件，点击提交发送院系审核。   

3、院系科研秘书进行审核，审核通过点击提交至知识产权管理员，不通过则退回科研人员。

4、科研人员确认科管系统审核通过后，携带专利负责人签字的委托代理合同等赋码材料前往科研院横向处（南校园中山楼101办公室）办理盖章。

5、科研人员将签订好的合同及有关附件电子版上传提交科管系统“盖章文件”，完成专利申请阶段审批后，可打印凭证，完成财务报销。

# 专利代理的核心工作是什么？
评估创新、撰写材料、流程申报

1. 检索创新性报告：新颖性、创造性、实用性评估。
2. 撰写与布局：专利权利要求书、说明书、附图。
3. 流程代理：提交、实审请求、无效应对、授权/年费管理。

**软著代理的核心工作是什么？****材料规范、合规检查、申报与补正**

1. 材料规范：版本号与权属证明；源代码节选（通常“前30页+后30页”且连续、可编译片段）、技术文档（功能/模块/流程图/接口/运行环境/界面截图）。
2. 合规检查：避免侵权与证据链断裂。
3. 申报与补正：在线填报、文件格式校验、补正说明、加急办理（如需）。

**何时宜用代理？**

+ **强烈建议（专利）：发明或核心实用新型、涉及算法+装置/医疗器械/嵌入式组合权利要求。**
+ **自行办理（软件著作权）：软件著作需求明确、代码与文档齐全、时间宽裕。**

**哪里可以找找到代理？**

可以询问老师学校官方代理，或者购物平台。但请注意甄别。
